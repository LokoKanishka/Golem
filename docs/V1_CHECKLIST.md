# V1 Checklist: Golem

## Objetivo
Definir cuándo Golem V1 puede considerarse estable y cerrado.

## Criterio general
V1 está cerrada cuando el sistema demuestra, de forma repetible, que cumple su función mínima sin depender de workers externos.

---

## 1. Superficie principal
- [ ] El panel del gateway funciona como consola principal
- [ ] La sesión principal de control se puede ubicar con claridad
- [ ] La conversación en panel mantiene continuidad básica

## 2. Canal auxiliar
- [ ] WhatsApp funciona como canal auxiliar/remoto
- [ ] Los mensajes por WhatsApp llegan y reciben respuesta
- [ ] La sesión de WhatsApp se refleja correctamente en el gateway
- [ ] WhatsApp no desplaza al panel como centro del sistema

## 3. Operación web mínima
- [ ] El browser relay por Chrome puede adjuntar una pestaña
- [ ] Golem puede detectar la pestaña correcta
- [ ] Golem puede leer el contenido accesible de la pestaña
- [ ] Golem puede resumir la pestaña de forma útil

## 4. Observación del sistema
- [ ] Se puede verificar que el gateway está vivo
- [ ] Se puede verificar que WhatsApp está conectado
- [ ] Se puede verificar que el browser relay está operativo
- [ ] Se pueden ubicar sesiones relevantes en el panel

## 5. Gobernabilidad
- [ ] El estado actual del sistema está reflejado en el repo
- [ ] La identidad de Golem está fijada
- [ ] Las fronteras de V1 están documentadas
- [ ] Las capacidades actuales/futuras/deseadas están separadas

## 6. Exclusiones confirmadas
- [ ] V1 no depende de Codex
- [ ] V1 no promete desktop agency completa
- [ ] V1 no promete pipeline de tareas con worker real
- [ ] V1 no mezcla artefactos complejos con conversación básica

---

## Condición de cierre
Golem V1 se considera cerrado cuando todos los puntos anteriores estén cumplidos o explícitamente aceptados como baseline real del sistema.
